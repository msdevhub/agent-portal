### MobileTimePicker padding 漂移修复 (23:12 CST)
- **根因**：CSS `padding: calc(50% - 32px)` — `%` padding 基于**元素宽度**不是高度。手机竖屏时容器窄但高，spacer padding 太小，高亮带 (`top:50%` 基于高度) 和 spacer 不一致 → 漂移
- **修复**：用 `useRef` + `clientHeight` 测量容器实际高度，`pad = floor(h/2 - ITEM_H/2)`，高亮带和 spacer 都用同一个 px 值
- **scroll-snap → JS snap**：去掉 CSS `scroll-snap`（iOS 上阻止滚到第一项），改用 JS debounced `scrollTo`

### 关键教训
- CSS `%` padding/margin 始终基于**包含块宽度**，不是高度！需要基于高度的居中必须用 JS 测量
