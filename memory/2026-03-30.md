### History API 时间精度 Bug 修复 (2026-03-29)
- **根因**：`/api/dashboard/history` 的 bot/server count 查询返回空数组。原因是 JS `toISOString()` 产出毫秒精度 (`938Z`)，但 Postgres 存储微秒精度 (`938667+00`)，`IN` 匹配失败
- **修复**：重写为单个 SQL（CTE + LEFT JOIN），在 DB 层直接完成 count，避免 JS↔DB 时间格式往返
- **结果**：summaries 正确返回 7 bots / 20-22 srvs

### DB 注意事项
- `exec_sql` 用于 DDL (CREATE/DROP)，`run_sql` 用于 DML (SELECT/INSERT with RETURNING)
- 建表必须双引号保持大小写：`"AP_daily_activities"` 否则会建成小写 `ap_daily_activities`
- `run_sql` 不支持不带 RETURNING 的 INSERT — 需用 `exec_sql`
