🎛️ PORTAL 指令栏 — 完整流程

1
👆 你点了 Bot 卡片上的 💬 按钮
比如点了 @craftbot 🏰 → 底部指令栏显示引用目标
2
⌨️ 你在指令栏输入文字
[@craftbot 🏰] 检查部署状态 → 点发送
3
🖥️ Portal 后端收到请求
POST /api/ops/send → 拿到 craftbot 的 user_id
4
💜 portalops → Mattermost API
用 portalops 的 token 给 @craftbot 发 DM："检查部署状态"
5
🦞 服务器上的龙虾收到消息
clawedbot 上的 @craftbot 龙虾 agent 收到 DM → 自动执行指令 → 回复结果
6
📨 回复回流到 Portal
前端轮询 GET /api/ops/messages → 读取 @craftbot 的回复 → 显示在指令栏上方

架构一览

👤 你（浏览器）
→
🖥️ Portal API
→
💜 Mattermost
→
🤖 @craftbot
→
🦞 龙虾执行
portalops bot 是中间的信使，负责在 Portal ↔ Mattermost 之间传话
🦞 执行结果
→
🤖 Bot 回复 DM
→
💜 Mattermost
→
🖥️ Portal 轮询
→
👤 你看到结果
回复反向流动：龙虾 → Bot DM → Portal 读取 → 页面显示
