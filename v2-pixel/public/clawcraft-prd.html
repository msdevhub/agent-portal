1. 产品定位

一句话

ClawCraft 是 OpenClaw 的 RTS 风格图形操作界面 — 用 2.5D 等距游戏世界替代传统的 Channel 对话和 CLI 配置工作。

核心价值（并列，不分主次）

价值 说明
监控 3 秒看懂全局 — "谁在干什么、是否异常"
操控 替代 Channel 对话 + CLI 配置 — 对话、指令、会话管理、记忆操作、技能安装、路由配置全在游戏界面完成
陪伴 长开当第二屏，有生活感的"数字鱼缸"
展示 OpenClaw 品牌资产，一看就懂 Agent 系统在干什么

设计原则：ClawCraft = OpenClaw 的图形化客户端。 用户不再需要打开 Telegram/Discord/Mattermost 跟 Agent 对话，也不需要用 CLI 改配置。所有交互都在 RTS 游戏世界里完成。

目标用户

主要用户: OpenClaw 管理者（如 @dora），同时运行多个 Agent、多个 Channel 的日常使用者。

2. ClawCraft 替代的传统工作流

传统方式 ClawCraft 操作 对应游戏元素
打开 Mattermost 发消息给 Agent 选中探险者 → 对话面板输入 领主大厅·信使台
在 Telegram 群里看 Agent 回复 观察卷轴飘出 + 单位卡·产出 Tab 发光卷轴
翻聊天记录看历史 左键档案馆 → Transcript 时间线 战史卷轴
openclaw gateway restart 双击王城 → 确认重启 王城操作
openclaw status 直接看地图 — 建筑亮灭、旗帜颜色 世界状态
手编 SOUL.md / AGENTS.md 右键大厅 → 编辑面板 大厅·文书阁
手编 openclaw.json bindings 拖旗到大厅 / 治理面板·路由图 拖拽连线
clawhub install <skill> 右键科技架 → 商店 → 安装 科技树界面
memory_store / recall / forget 右键图书馆 → 操作面板 图书馆交互
切换不同 Channel 看不同 Agent 全在一张地图上，镜头移动即切换 地图导航
查看 Token 用量 HUD 资源条 + 历史图表 粮草金币

3. 视觉设计

风格

2.5D 等距像素风（Isometric Pixel Art），致敬帝国时代 2。赛博中世纪色调：石砌建筑 + 服务器蓝光。

地图布局

固定等距大地图，中心是王城（Gateway），周围分布各 Agent 的领地。

              [前哨塔-Node]
                    |
    [港口-Channel] ─── [王城-Gateway] ─── [港口-Channel]
                          |
     [Agent领地-A]    [Agent领地-B]    [Agent领地-C]
      ├ 大厅              ├ 大厅            ├ 大厅
      ├ 图书馆(Memory)    ├ 作坊群(Tool)   ├ ...
      ├ 作坊群(Tool)      ├ 科技架(Skill)
      ├ 碑塔(Compaction)  └ 档案馆(Transcript)
      └ 科技架(Skill)

素材策略

  • MVP：简单几何体 + 颜色（白模），验证后再换精细素材
  • AI 辅助概念图，统一后制作 spritesheet
  • 所有素材需 CC0/MIT 许可
  • 未启用/未配置系统: 显示为半透明蓝图（不隐藏），提示用户可以建造

开发者滤镜（Developer Vision）

一键切换：

  • 游戏模式：完整 RTS 视觉隐喻
  • 开发者模式：实体显示真实技术标签，建筑简化为带状态色方块

4. 核心概念映射 + 实现规格

每个概念包含: 地图表现 / 面板 Tab 设计 / 操作安全等级 / 真实 API

4.1 Gateway（王城 🏰）

OpenClaw 能力: 中央服务进程。管理所有 Agent、Channel、Plugin。openclaw status/gateway restart/stop/start。配置 bind/port/auth。openclaw doctor --fix 诊断修复。日志 /tmp/openclaw-<uid>/openclaw-<date>.log。

地图表现:

  • 中央堡垒 + 全图光圈
  • 绿=运行 / 黄=重启中 / 红=停止
  • 城墙上的旗帜数 = 在线 Channel 数

王城面板 (5 Tab):

Tab 内容 API
概览 版本 / 运行时长 / bind模式 / 端口 / Agent数 / Session数 / Plugin数 GET /clawcraft/state
健康 各 channel 连接状态、health-monitor 指标 GET /clawcraft/config → health-monitor 段
配置 gateway 核心配置编辑（bind/port/auth） POST /clawcraft/action { type: "gateway.config" }
插件 已加载插件列表（名称/状态/版本），启用/禁用 openclaw plugins list/enable/disable
日志 最近 100 行日志，可按级别(info/warn/error)过滤 tail -100 /tmp/openclaw-*/openclaw-*.log

操作:

操作 安全 触发 API
查看状态 🟢 左键王城 GET /clawcraft/state
查看配置 🟢 王城面板·配置Tab GET /clawcraft/config
重启 Gateway 🟠 双击王城 → 确认弹窗 POST /clawcraft/action { type: "gateway.restart" }
诊断修复 🟡 王城面板·健康Tab → "诊断" openclaw doctor --fix

4.2 Agent（领主大厅 🏛️）

OpenClaw 能力: 独立人格的 AI 实体。有自己的 SOUL.md/AGENTS.md/TOOLS.md、model、skills、workspace。

地图表现:

  • 独立城池 + 英雄旗帜
  • 旗色=Agent persona / 冒烟=有活跃 Session
  • 大厅旁建筑群 = 该 Agent 的工具/技能/记忆

领主大厅面板 (5 Tab):

Tab 内容 API
概览 名称 / 模型 / 状态 / 活跃session数 / soulSummary GET /clawcraft/state → agents
对话 ChatPanel — 选 session 或新建，发消息看回复 POST /clawcraft/chat/send
文件 SOUL.md / AGENTS.md / TOOLS.md / IDENTITY.md 编辑器 GET/POST /clawcraft/files?agentId=xxx
记忆 搜索/添加/删除记忆 POST /clawcraft/memory
技能 已装 Skill 列表 / 安装新 Skill GET/POST /clawcraft/skills

操作:

操作 安全 触发 API
查看 🟢 左键大厅 GET /clawcraft/state
对话 🟢 双击大厅 / 面板·对话Tab POST /clawcraft/chat/send
编辑 SOUL.md 🟡 面板·文件Tab POST /clawcraft/files
新建 Session 🟡 右键 → 新建对话 POST /clawcraft/action { type: "session.new" }

4.3 Session（探险者 ⚔️）

OpenClaw 能力: 一次对话上下文。有 sessionId、sessionKey、transcript。状态：idle/thinking/tooling/responding/ended。

地图表现:

  • 单兵单位在大厅周围活动
  • 血条 = context 剩余量
  • 状态颜色: 蓝=思考 / 橙=工具 / 白=发言 / 灰=idle / 红=错误

探险者面板 (4 Tab):

Tab 内容 API
概览 status / model / runCount / toolCallCount / errorCount GET /clawcraft/state → sessions
对话 ChatPanel — 当前 session 消息流 POST /clawcraft/chat/send
时间线 Transcript 纵向时间轴 (user/assistant/tool 节点) GET /clawcraft/chat/history?sessionKey=xxx
控制 压缩(compact) / 重置(reset) / 切换模型 POST /clawcraft/action

操作:

操作 安全 API
查看 🟢 GET /clawcraft/state
发消息 🟢 POST /clawcraft/chat/send
压缩对话 🟡 POST /clawcraft/action { type: "session.compact" }
重置会话 🟠 POST /clawcraft/action { type: "session.reset" }

4.4 Channel（港口使馆 ⚓）

OpenClaw 能力: 22 种消息频道（WhatsApp/Telegram/Discord/Mattermost/Signal/Slack/IRC/LINE/Matrix 等）。配置维度: dmPolicy(pairing/allowlist/open/disabled), groupPolicy(allowlist/open/disabled), allowFrom[], accounts(多账号), chatmode(oncall/onmessage/onchar)。路由: channel → account → agent (通过 bindings)。

地图表现:

  • 港口建筑在地图边缘，每种频道有专属旗帜
  • 状态灯：绿=在线，灰=离线/禁用，黄=连接中，红=错误
  • 路由线：港口→Agent 大厅的发光连线，表示 bindings

港口面板 (4 Tab):

Tab 内容 API
概览 频道类型 / 状态 / 账号名 / dmPolicy / groupPolicy GET /clawcraft/config → channels
配置 dmPolicy / groupPolicy / allowFrom / chatmode / mediaMaxMb 编辑 POST /clawcraft/action { type: "channel.update" }
路由 当前 binding（绑到哪个 Agent）+ 修改 POST /clawcraft/action { type: "binding.update" }
日志 最近消息流量（收/发计数 + 最后消息时间） SSE 事件统计

操作:

操作 安全 API
查看状态 🟢 GET /clawcraft/config
修改配置 🟡 POST /clawcraft/action { type: "channel.update" }
添加频道 🟠 见场景 BIZ-03 向导流程
删除频道 🟠 POST /clawcraft/action { type: "channel.remove" }
改路由 🟠 POST /clawcraft/action { type: "binding.update" }

4.5 Skill（科技卷轴 📜）

OpenClaw 能力: workspace/skills/ 下的文件夹 + SKILL.md。来源: ClawHub 商店 (clawhub CLI) 或手动。管理: clawhub search/install/list/update/publish。

地图表现:

  • 科技架建筑群在 Agent 大厅旁
  • 已装 Skill = 发光书架节点，未装 = 虚影蓝图

科技树面板 (2 Tab):

Tab 内容 API
已建造 已装 Skill 列表（名称/版本），可卸载 GET /clawcraft/skills?action=list
探索商店 搜索 ClawHub，一键安装 GET /clawcraft/skills?action=search&q=xxx → POST { action: "install" }

4.6 Cron/Heartbeat（钟楼 🕐）

OpenClaw 能力: Gateway 内置调度器。持久化 ~/.openclaw/cron/jobs.json。三种调度: at(一次性) / every(固定间隔) / cron(cron表达式+时区)。两种 session 模式: main(主session) / isolated(独立session)。三种交付: announce(回到 channel) / webhook / none。CLI: openclaw cron add/list/edit/remove/run/runs。

地图表现:

  • 钟楼在王城旁，摆钟动画
  • Cron 执行时：钟楼敲响 + 探险者出发
  • 角标 = 活跃 job 数

钟楼面板 (3 Tab):

Tab 内容 API
任务列表 所有 jobs（名称/调度/上次运行/状态），启用/禁用/删除 openclaw cron list
新建任务 见场景 BIZ-05 向导流程 openclaw cron add
运行日志 run 记录（时间/耗时/状态/输出预览） openclaw cron runs --id <id>

4.7 Node（前哨塔 🗼）

OpenClaw 能力: 配对设备 (iOS/Android/macOS/headless)，WebSocket 连接 Gateway。能力: camera.snap/camera.clip (拍照/录视频)、screen.record (录屏)、device.status/info/permissions/health (设备信息)、notifications.list/action (通知管理)、location.get (位置)、system.run (远程命令)、canvas.present (展示UI)。配对: 设备发起 → Gateway 创建 request → 用户批准。

地图表现:

  • 前哨塔在地图边缘，每个 Node 一座
  • 设备图标：📱iOS / 🤖Android / 💻macOS
  • 心跳脉冲动画 / 离线=灰色

前哨塔面板 (4 Tab):

Tab 内容 API
概览 设备名 / 类型 / OS版本 / IP / 在线时长 nodes status + nodes describe
能力 camera/screen/location/notifications 各项启用状态 nodes describe
操作 拍照/截屏/获取位置/发通知 快捷按钮 nodes camera_snap/screen_record/location_get
配对 待审批设备列表 + 批准/拒绝 nodes pending/approve/reject

4.8 Plugin（齿轮工坊 ⚙️）

OpenClaw 能力: openclaw plugins install/list/enable/disable。来源: stock(bundled)/npm/local。配置: plugins.allow[], plugins.deny[], plugins.entries.<id>.enabled。Manifest: openclaw.plugin.json。

地图表现: 齿轮工坊在王城内部。已加载=亮齿轮转动，禁用=暗灰静止。

齿轮工坊面板 (3 Tab):

Tab 内容 API
已装 插件列表（名称/ID/状态/来源），启用/禁用 openclaw plugins list
安装 路径或 npm 包名输入 → 安装 openclaw plugins install <spec>
配置 插件 configSchema 渲染为表单 待实现

4.9 Tool（作坊群 🔨）

OpenClaw 能力: 20+ 内置工具。6 分类: gather(web_search/fetch/browser), observe(read), build(write/edit), forge(exec/process), memory(memory_*), message(message/tts), spawn(sessions_spawn)。策略: tools.allow/deny, tools.byProvider。

地图表现:

  • 作坊群在 Agent 大厅周围，按分类聚合（铁匠铺=forge, 风车=gather, 图书馆=memory）
  • 工具执行时：对应作坊亮起
  • 禁用工具 = 锁图标

4.10 Memory（图书馆 📚）

OpenClaw 能力: memory_search(搜索), memory_store(存储), memory_forget(删除), memory_update(更新)。通过 /tools/invoke 调用。支持 memory-core(FTS) 和 memory-lancedb-pro(Vector)。

地图表现: 书库建筑。热记忆发光，旧记忆蒙尘。

图书馆面板 (3 Tab):

Tab 内容 API
搜索 关键词搜索，结果列表(预览200字) POST /clawcraft/memory { action: "recall" }
存储 输入文本+分类+重要度 → 存储 POST /clawcraft/memory { action: "store" }
管理 修改/删除记忆 POST /clawcraft/memory { action: "update/forget" }

4.11 Compaction（编年碑 🪦）

OpenClaw 能力: context window 快满时自动触发，或手动 /compact。压缩长对话为摘要。

地图表现: 石碑造型，碑上刻痕数 = compaction 次数。动画：僧侣出场 → 卷轴汇流 → 刻入碑塔。

4.12 Transcript（战史卷轴 📖）

OpenClaw 能力: JSONL 文件。sessions_history(sessionKey, limit, includeTools)。

地图表现: 档案馆建筑。新消息写入时墨迹高亮。

4.13 Sub-Agent（雇佣英雄 🗡️）

OpenClaw 能力: sessions_spawn(task, agentId, mode, runtime, model)。模式: run(一次性)/session(持久)。运行时: subagent(内部)/acp(外部coding agent)。管理: subagents list/steer/kill。

地图表现: 从兵营召唤出场。深度徽章(L1/L2/L3)。链标连线: sub→parent。完成后回城交付。

5. 三大系统层（辩论共识必补）

以下 3 个跨对象系统层是场景辩论中 Gemini + GPT-5.4 共识的最大缺陷。

5.1 Global HUD / Ops Center（全局运维中心）

问题: 10 个概念各自有面板，但用户打开 ClawCraft 的第一需求是"3 秒看懂全局"，目前无法做到。

实现规格:

A. Morning Brief 弹窗（每次打开/重连时显示）:

┌─────────────────────────────────────────────┐
│  ☀️ 早安，离线期间发生了:                      │
│  ──────────────────────────────────────────  │
│  ✅ 3 个 Cron 任务成功执行                     │
│  ⚠️ 1 个频道断连后已自动恢复 (Telegram 03:22)  │
│  ❌ 1 个错误: coder 限流 429 (已恢复)          │
│  📊 Token 消耗: 12,340 (research 8,200)      │
│  🧠 2 次 Compaction (research, main)          │
│  ──────────────────────────────────────────  │
│         [查看详情]        [关闭]              │
└─────────────────────────────────────────────┘

数据来源:

  • SSE 事件缓存 (plugin StateManager 维护最近 24h 摘要)
  • API: GET /clawcraft/state 扩展 overnightSummary 字段

B. HUD 健康指标条（始终显示在地图顶部）:

🏰 运行中 | 🏛️ 3 Agent | ⚔️ 9 Session(7活跃) | ⚓ 4 Channel(3在线) | 🗼 1 Node | ❌ 0 错误
  • 错误数 > 0 时红色闪烁
  • 点击任何指标 → 跳转到对应实体列表

C. 地图宏观状态透出:

  • 建筑头顶状态气泡: ✅ 正常 / ⚠️ 警告 / ❌ 错误 / 💤 空闲
  • 不需要点击就能从地图看出哪里有问题

5.2 Incident Center（事故处理中心）

问题: 429 限流、Channel 断连、Memory 满等事故是跨对象的，不能靠各建筑分散处理。

实现规格:

A. 事件模型:

interface Incident {
  id: string;
  severity: 'info' | 'warning' | 'error' | 'critical';
  source: { type: 'agent' | 'channel' | 'gateway' | 'node' | 'cron'; id: string };
  title: string;           // 人类可读标题
  detail: string;          // 技术细节
  firstSeen: number;       // 首次发现时间
  lastSeen: number;        // 最后发生时间
  count: number;           // 累计次数
  status: 'open' | 'acked' | 'resolved' | 'muted';
  blastRadius: string[];   // 影响范围 ["agent:research", "cron:morning-brief"]
  suggestedActions: SuggestedAction[];
}

interface SuggestedAction {
  label: string;           // "重启 Gateway" / "切换模型" / "清理 Memory"
  actionType: string;      // "gateway.restart" / "session.compact"
  params: Record<string, any>;
  safety: '🟢' | '🟡' | '🟠';
}

B. 事故类型 + 检测 + 修复:

事故类型 检测方式 地图表现 建议操作
429 限流 llm_output hook 返回 429 Agent 大厅头顶红色沙漏 🟡 暂停 cron / 🟡 切换备用模型
Channel 断连 health-monitor / SSE 无心跳 港口红灯 + 路由线断裂 🟡 重新连接 / 🟡 检查 token
Context 满 session context_usage > 90% 探险者血条红 + 警告气泡 🟡 压缩对话
Gateway 崩溃 SSE 断开 + 重连失败 全图灰化 + 王城冒烟 🟠 重启 Gateway
Sub-Agent 失控 spawn 计数 > 阈值 (如 >10) 兵营溢出 + 红色警报 🟠 一键终止所有 Sub-Agent
Cron 失败 cron run 状态 error 钟楼红灯 🟢 查看日志 / 🟡 立即重跑
Plugin 异常 plugin 报错日志 齿轮工坊冒烟 🟡 禁用插件 / 🟠 重启 Gateway

C. 事故面板 UI:

┌──────────────────────────────────────────────────┐
│  🚨 事故中心          [全部] [未处理] [已静音]    │
│  ────────────────────────────────────────────────│
│  ❌ CRITICAL  coder Agent 429 限流                │
│     影响: coder agent, 2 session                  │
│     持续: 15 分钟 (首次 14:22)                    │
│     [切换模型🟡] [暂停 cron🟡] [标记已处理]       │
│  ────────────────────────────────────────────────│
│  ⚠️ WARNING  Telegram 频道连接不稳定              │
│     影响: telegram-bot-1 → research agent         │
│     持续: 3 分钟 (已自动重连 2 次)                │
│     [重新连接🟡] [检查配置🟢] [静音 1h]           │
│  ────────────────────────────────────────────────│
│  ℹ️ INFO  research agent compaction 完成          │
│     影响: session main, 上下文从 95% → 42%        │
│     [查看详情🟢] [标记已阅]                       │
└──────────────────────────────────────────────────┘

触发方式:

  • HUD 错误计数 > 0 → 右下角浮窗自动弹出

BIZ-01: 首次打开 ClawCraft

触发: 用户首次访问 craft.dora.restry.cn

步骤:

  1. 浏览器加载 → GET /clawcraft/state 获取初始状态

  5. 未配置/未启用的概念显示半透明蓝图 + "点击建造" 提示文字

API: GET /clawcraft/state 需扩展返回:

{

状态模型

interface WorldState {
  // 基础
  gateway: GatewayState;
  agents: AgentState[];
  sessions: SessionState[];
  
  // 新增: 运维
  channels: Record<string, ChannelStatus>;
  incidents: Incident[];
  overnightSummary?: OvernightSummary;
  onboardingProgress?: OnboardingProgress;
}

interface SessionState {
  sessionKey: string;
  sessionId: string;
  agentId: string;
  status: 'idle' | 'thinking' | 'tooling' | 'responding' | 'blocked' | 'ended';
  currentTool?: string;
  currentToolCategory?: ToolCategory;
  lastAssistantPreview?: string;
  runCount: number;
  toolCallCount: number;
  errorCount: number;
}

interface AgentState {
  agentId: string;
  name: string;
  model: string;
  status: 'online' | 'offline' | 'unknown';
  soulSummary?: string;
  toolNames: string[];
  skillIds: string[];
  sessionKeys: string[];
}

interface Incident {
  id: string;
  severity: 'info' | 'warning' | 'error' | 'critical';
  source: { type: string; id: string };
  title: string;
  detail: string;
  firstSeen: number;
  lastSeen: number;
  count: number;
  status: 'open' | 'acked' | 'resolved' | 'muted';
  blastRadius: string[];
  suggestedActions: { label: string; actionType: string; params: any; safety: string }[];
}

断线恢复

  1. SSE 自动重连 + 指数退避（1s→2s→4s→8s→30s max）
  2. 重连后 GET /clawcraft/state 全量恢复
  3. serverInstanceId 变化 → 清空队列重建世界
  4. HUD 显示 Reconnecting + 世界灰化
